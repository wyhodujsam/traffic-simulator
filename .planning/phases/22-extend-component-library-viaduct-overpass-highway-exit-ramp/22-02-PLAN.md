<interfaces>
ComponentSpecDto has these Jackson-populated fields:
```java
public String type;
public String id;
public Point2D.Double centerPx;
public Point2D.Double startPx;   // (unused by Viaduct/Ramp)
public Point2D.Double endPx;     // (unused)
public double rotationDeg;
public List<String> armsPresent; // (unused by Viaduct/Ramp — arms are fixed)
public Double lengthPx;          // (unused)
```

Both new records have fixed arms, so the DTO's `armsPresent` is ignored for VIADUCT and
HIGHWAY_EXIT_RAMP. The record constructors take (id, center, rotationDeg) only.

Existing toSpec shape (switch on d.type, returns ComponentSpec):
```java
case "ROUNDABOUT_4ARM" -> new RoundaboutFourArm(d.id, d.centerPx, d.rotationDeg, d.armsPresent);
...
default -> throw new ClaudeCliParseException("Unknown component type: " + d.type + ...);
```
</interfaces>
