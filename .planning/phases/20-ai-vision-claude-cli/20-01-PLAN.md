  <action>
1. Create `ClaudeCliConfig.java` in config package:
   - `@ConfigurationProperties(prefix = "claude.cli")`
   - `@Component`
   - Fields: `String path` (default: "claude"), `int timeoutSeconds` (default: 30), `String tempDir` (default: System.getProperty("java.io.tmpdir"))
   - Register in application.properties: `claude.cli.path=claude`, `claude.cli.timeout-seconds=30`

2. Create `ClaudeVisionService.java` in service package:
   - `@Service`, `@RequiredArgsConstructor`, `@Slf4j`
   - Inject: `ClaudeCliConfig config`, `MapValidator mapValidator`, `ObjectMapper objectMapper`
   - Method: `public MapConfig analyzeImage(MultipartFile file) throws IOException`
   - Implementation:
     a. Save the uploaded file to a temp file: `Files.createTempFile(Path.of(config.getTempDir()), "vision-", getExtension(file.getOriginalFilename()))`, then `file.transferTo(tempFile)`
     b. Build the Claude CLI prompt — a detailed prompt asking Claude to analyze the road image and output a MapConfig JSON. The prompt MUST specify the exact MapConfig schema (nodes, roads, intersections, spawnPoints, despawnPoints) with field names matching MapConfig.java. Include: "Output ONLY valid JSON, no markdown fences, no explanation text."
     c. Execute via ProcessBuilder: `new ProcessBuilder(config.getPath(), "-p", prompt, "--image", tempFile.toAbsolutePath().toString())`. Redirect stderr to stdout. Set timeout via `process.waitFor(config.getTimeoutSeconds(), TimeUnit.SECONDS)`.
     d. Read stdout fully into a String.
     e. Delete temp file in finally block.
     f. If process timed out (waitFor returns false): `process.destroyForcibly()`, throw `ClaudeCliTimeoutException("Analysis timed out after " + config.getTimeoutSeconds() + " seconds")`
     g. If exit code != 0: throw `ClaudeCliException("Claude CLI failed with exit code " + exitCode + ": " + stdout)`
     h. Extract JSON from stdout — look for first `{` and last `}` to handle any preamble/postamble text. If no JSON found, throw `ClaudeCliParseException("Could not find JSON in Claude CLI output")`
     i. Parse extracted JSON string into MapConfig using objectMapper.readValue(). Catch JsonProcessingException and throw `ClaudeCliParseException("Invalid JSON: " + e.getMessage())`
     j. Validate with mapValidator.validate(). If errors non-empty, throw `ClaudeCliParseException("Generated config has validation errors: " + String.join(", ", errors))`
     k. Return valid MapConfig.
   - The prompt should instruct Claude to:
     - Identify roads, lanes, intersections from the image
     - Generate unique node IDs (n1, n2, n3...)
     - Generate road IDs (r1, r2, r3...)
     - Estimate road lengths in pixels (100-500 range)
     - Set speedLimit to 50.0 (default urban)
     - Detect lane count from visual cues (default 2)
     - Add spawnPoints at entry nodes, despawnPoints at exit nodes
     - Set id to "vision-generated" and name to "AI Generated Map"

3. Create custom exception classes as static inner classes of ClaudeVisionService (or separate files — keep it simple):
   - `ClaudeCliException extends RuntimeException`
   - `ClaudeCliTimeoutException extends RuntimeException`
   - `ClaudeCliParseException extends RuntimeException`

4. Helper: `private String getExtension(String filename)` — returns ".png" or ".jpg" based on filename, default ".png".
