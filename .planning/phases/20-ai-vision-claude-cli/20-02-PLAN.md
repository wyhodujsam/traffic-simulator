1. In `MapPage.tsx`, add `handleUploadImage` callback:
   - Accepts a `File` parameter.
   - Sets `sidebarState` to `'loading'`, clears errors.
   - Creates a `FormData`, appends the file as `"image"`.
   - POSTs to `/api/vision/analyze` with the FormData (NO Content-Type header — browser sets multipart boundary automatically).
   - On success: parse response as MapConfigData, set `mapConfig`, set `fetchResult` with road/intersection counts, set `sidebarState` to `'result'`.
   - On error: parse error JSON for `error` field, set `fetchError`, set `sidebarState` to `'error'`.
   - Wrap in useCallback with no deps (only uses setters).
   - Pass `onUploadImage={handleUploadImage}` to MapSidebar.

2. In `MapSidebar.tsx`:
   - Add `readonly onUploadImage?: (file: File) => void;` to MapSidebarProps.
   - Replace the disabled "Upload Image" button in `IdleActions` with an enabled button.
   - Add `onUploadImage` prop to `IdleActions`.
   - On "Upload Image" click: create a hidden `<input type="file" accept="image/jpeg,image/png">` programmatically, trigger click, on change call `onUploadImage(file)`.
   - Implementation: use a ref to a hidden file input OR create dynamically. The simplest approach:
     ```tsx
     const fileInputRef = useRef<HTMLInputElement>(null);
     // In JSX:
     <input type="file" ref={fileInputRef} accept="image/jpeg,image/png"
       style={{ display: 'none' }}
       onChange={(e) => { if (e.target.files?.[0] && onUploadImage) onUploadImage(e.target.files[0]); }} />
     <button style={buttonBase} onClick={() => fileInputRef.current?.click()}>
       Upload Image
     </button>
     ```
   - When in `loading` state (triggered by upload), show "Analyzing road image..." instead of "Fetching road data..." in LoadingContent. To differentiate, add an optional `loadingMessage` prop to MapSidebarProps, default "Fetching road data...". MapPage passes `loadingMessage="Analyzing road image..."` when upload is in progress vs. OSM fetch.
   - Pass `onUploadImage` through from MapSidebar to IdleActions component.

3. In `MapPage.tsx`, add state to track whether we're in upload or OSM mode:
   - `const [uploadMode, setUploadMode] = useState(false);`
   - In handleUploadImage: `setUploadMode(true)` before setting loading.
   - In handleFetchRoads: `setUploadMode(false)` before setting loading.
   - Pass `loadingMessage={uploadMode ? 'Analyzing road image...' : 'Fetching road data...'}` to MapSidebar.

4. Result state reuse: After upload succeeds, the same ResultContent component shows "Roads loaded" with road/intersection counts, plus Run Simulation, Export JSON, New Selection buttons. These already work because they operate on `mapConfig` state which is set by both OSM fetch and image upload paths. No changes needed to ResultContent.

5. Error state reuse: ErrorContent already shows the error message and "Try Again" button. The `onReset` handler already clears state back to idle. No changes needed.
