# Plan 12.3: Frontend — clip road rendering at intersection boundaries

Modify `drawRoads.ts` to trim road rendering by `clipStart`/`clipEnd` pixels. Roads still have the same coordinate endpoints, but the visible portion starts/ends outside the intersection zone. Lane markings, boundaries, and center lines must not extend into the clipped zone.

## Task 12.3.1: Trim road fill and markings by clipStart/clipEnd

<read_first>
- /home/sebastian/traffic-simulator/frontend/src/rendering/drawRoads.ts
- /home/sebastian/traffic-simulator/frontend/src/rendering/constants.ts
- /home/sebastian/traffic-simulator/frontend/src/types/simulation.ts
</read_first>

<action>
1. In `drawRoad()` function, after computing `roadLength` and `angle`, calculate the visible segment:

```typescript
const clipS = road.clipStart ?? 0;
const clipE = road.clipEnd ?? 0;
const visibleStart = clipS;
const visibleLength = roadLength - clipS - clipE;
if (visibleLength <= 0) return;  // entire road is inside intersection
```

2. Change the road fill rectangle from:
```typescript
ctx.fillRect(0, -roadWidth / 2, roadLength, roadWidth);
```
to:
```typescript
ctx.fillRect(visibleStart, -roadWidth / 2, visibleLength, roadWidth);
```

3. Apply the same clipping to ALL road markings:

**Road boundaries** (solid white lines): draw from `visibleStart` to `visibleStart + visibleLength` instead of `0` to `roadLength`.

**Lane dividers** (white dashed): same x range.

**Yellow center line** (paired roads): same x range.

**Closed lane hatching**: clip the hatching fill and diagonal lines to the visible range.

4. Keep the `ctx.translate(road.startX, road.startY)` and `ctx.rotate(angle)` unchanged — the coordinate system is still road-local, we just start drawing further along.
</action>
