## Decisions Made

- **Override removed, not just commented out.** The plan-checker's iron-law in Plan 24.2-01 — "the resolver returns the configured path UNCHANGED so callers see clean ENOENT — never silently fabricate a phantom path" — applies to test infrastructure too: leaving the override in place would let the e2e suite continue passing if someone reverts the resolver, masking the regression. Removed deliberately so a future resolver regression FAILS the real-backend e2e spec.
- **Comment block updated, not deleted.** The "why jvmArguments not arguments" rationale is still relevant (composes cleanly with future overrides; comma-split pitfall documented). The Phase 24.2 note explaining what the removed override used to do is paper trail for future readers — they see WHY the override is missing, not just that it's gone.
- **Single-quote quoting style preserved.** Even with one `-D` entry remaining, `'-Dspring-boot.run.jvmArguments=…'` is the same shape; future additions slot in without re-discovering the comma-split rule.
- **Phase 24.1 contract preserved verbatim.** `osm.overpass.urls=http://localhost:18086` was Phase 24.1 Plan 03's deliverable — redirects OverpassXmlFetcher to the local fixture server (port 18086) so the real-backend spec runs hermetically without hitting overpass-api.de. That override is unrelated to the binary path and stays.
