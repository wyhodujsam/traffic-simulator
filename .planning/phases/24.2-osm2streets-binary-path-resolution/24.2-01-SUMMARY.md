# Phase 24.2 Plan 01: Osm2StreetsConfig Binary-Path Resolution Summary

**Smart resolver in `Osm2StreetsConfig.getBinaryPath()` (`.git`-only project-root walk-up) closes the working-directory gap that returned HTTP 503 for `POST /api/osm/fetch-roads-o2s` when the backend was started with `cwd=backend/`; landed via TDD discipline (RED → GREEN → real-repo-smoke) with 5 @TempDir scenario tests + 1 real-repo smoke test, no Mockito anti-patterns, full backend suite 390/0/0/1.**

## The 4-step resolution algorithm landed in `getBinaryPath()`

```
1. binaryPath absolute → return unchanged                           [AC#3]
2. cwd-relative resolves to existing file → return unchanged        [AC#4]
3. walk up from user.dir → first ancestor with `.git` entry
   (file OR directory; supports git-worktrees) →
   if projectRoot.resolve(binaryPath) exists as regular file →
   return absolute path                                             [AC#1]
4. otherwise → return binaryPath unchanged so callers see ENOENT
```

The resolver runs INSIDE the getter so `Osm2StreetsService.executeCli`
(line 166: `new ProcessBuilder(config.getBinaryPath())`) and
`Osm2StreetsService.isAvailable` (line 106:
`Files.isExecutable(Path.of(config.getBinaryPath()))`) see the resolved
value transparently — no service-side changes required.

## Project-root marker decision: `.git`-only

**Chosen:** `.git` ONLY as the project-root marker.

**Verified-filesystem evidence (live repo, plan-checker iter-1):**
- `/home/sebastian/apps/traffic-simulator/.git` — EXISTS (directory)
- `/home/sebastian/apps/traffic-simulator/pom.xml` — DOES NOT EXIST
- `/home/sebastian/apps/traffic-simulator/backend/pom.xml` — exists (only POM in repo)
- `/home/sebastian/apps/traffic-simulator/backend/.git` — DOES NOT EXIST

### Dismissed alternatives

- **Combined `.git` + `pom.xml` (iter-0 design)** — REJECTED. The real
  repo has no top-level `pom.xml`. `findProjectRoot` would return `null`
  and the resolver would return the unchanged buggy path — leaving the
  bug unfixed. Witnessed empirically in the counterfactual sanity check
  below.
- **`pom.xml`-only** — REJECTED. `backend/pom.xml` would falsely match;
  walking up from `cwd=backend/` would stop at `backend/` and the
  resolver would compute `backend/.resolve("backend/bin/...")` →
  `backend/backend/bin/...` → ENOENT. Same bug.
- **Default-flip to `bin/osm2streets-cli-linux-x64` (CONTEXT.md option c)** —
  REJECTED. Would have worked but every test that runs from
  project-root cwd then needs adjustment OR an explicit override.
  Option (a) is more robust and only changes the resolver code.

## Decisions Made

- **Project-root marker = `.git` only** (not combined with `pom.xml`).
