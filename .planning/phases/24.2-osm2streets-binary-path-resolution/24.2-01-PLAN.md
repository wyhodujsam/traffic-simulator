<objective>
Fix `Osm2StreetsConfig.getBinaryPath()` so the configured path resolves to an existing binary regardless of the JVM's working directory. The current default `backend/bin/osm2streets-cli-linux-x64` only resolves correctly when cwd=project-root; when cwd=`backend/` (the documented dev flow `cd backend && mvn spring-boot:run`), ProcessBuilder produces `backend/backend/bin/...` which does not exist, and `POST /api/osm/fetch-roads-o2s` returns 503.

Strategy chosen from 24.2-CONTEXT.md decision space: **Option (a) — smart resolver inside `Osm2StreetsConfig.getBinaryPath()`**.
Algorithm (executed each call; no caching to keep test isolation simple):

1. Take `binaryPath` as configured (the raw field value).
2. Convert to a `Path`. If it is absolute → return as-is unchanged. (Backwards-compat AC#3.)
3. If the relative path resolved against `user.dir` exists as a regular file → return the configured `binaryPath` string unchanged. (Backwards-compat for Phase 24.1's test override and for production-style invocations from project root.)
4. Otherwise, walk up from `user.dir` looking for a "project root marker" (the first ancestor directory that contains BOTH a `.git` entry AND a `pom.xml` — handles git-worktrees and non-Maven submodules without false positives). Stop at filesystem root.
5. If a project root is found AND `projectRoot.resolve(binaryPath)` exists as a regular file → return that absolute path as a String.
6. Otherwise → return the configured `binaryPath` string unchanged. The call site (`executeCli` / `isAvailable`) will then surface a clean ENOENT-style error rather than the resolver hiding the failure with a phantom path.

Purpose: close the working-directory gap captured in 24.1-02-SUMMARY.md Rule-3 deviation #3 and 24.2-CONTEXT.md sections "Bug summary" + "Why Phase 24.1 didn't catch it". After this plan ships, no `-Dosm2streets.binary-path` override is needed for either dev-flow invocation style, and absolute / explicit-override paths keep working unchanged.
Output: `Osm2StreetsConfig.java` carrying the resolver + a new `Osm2StreetsConfigTest.java` with 5 scenarios that prove the resolver behavior directly (no Mockito; uses `@TempDir` for filesystem fixtures per `mockito-anti-patterns.md` rules).

       ```java
       /**
        * Returns the path to the osm2streets-cli binary, smart-resolved against
        * the current working directory.
        *
        * <p>Resolution algorithm (24.2-CONTEXT.md §"Decision space" option (a)):
        * <ol>
        *   <li>If {@code binaryPath} is absolute → return it unchanged.</li>
        *   <li>If {@code Path.of(user.dir).resolve(binaryPath)} exists as a regular
        *       file → return {@code binaryPath} unchanged (cwd-relative form works).</li>
        *   <li>Walk up from {@code user.dir} looking for a directory that contains
        *       BOTH {@code .git} AND {@code pom.xml}. If found and
        *       {@code projectRoot.resolve(binaryPath)} exists as a regular file →
        *       return that absolute path as a String.</li>
        *   <li>Otherwise → return {@code binaryPath} unchanged. The caller's
        *       ProcessBuilder will then surface a clean ENOENT-style error rather
        *       than this resolver hiding the failure with a phantom path.</li>
        * </ol>
        *
        * <p>Step (1) preserves Phase 24.1's absolute-path overrides (24.2-CONTEXT.md
        * AC#3). Step (2) preserves the OsmPipelineSmokeIT cwd-relative override
        * style (AC#4). Step (3) is the actual fix — handles
        * {@code cd backend && mvn spring-boot:run} (AC#1). Step (4) keeps the
        * iron-law: failure surfaces, never silently rewrites to a phantom path.
        */
       public String getBinaryPath() {
           if (binaryPath == null) {
               return null;
           }
           Path raw = Paths.get(binaryPath);
           if (raw.isAbsolute()) {
               return binaryPath;
           }
           Path cwd = Paths.get(System.getProperty("user.dir"));
           if (Files.isRegularFile(cwd.resolve(raw))) {
               return binaryPath;
           }
           Path projectRoot = findProjectRoot(cwd);
           if (projectRoot != null) {
               Path resolved = projectRoot.resolve(raw);
               if (Files.isRegularFile(resolved)) {
                   return resolved.toString();
               }
           }
           return binaryPath;
       }

       /** Test seam: returns the as-configured path, bypassing resolution. */
       String getRawBinaryPath() {
           return binaryPath;
       }

       /**
        * Walks up from {@code start} returning the first ancestor that contains
        * BOTH {@code .git} (file or directory — supports git-worktrees) AND
        * {@code pom.xml}. Returns {@code null} if no such ancestor exists.
        * Requiring BOTH markers prevents false positives in
        * {@code backend/} (which has its own pom.xml but no .git).
        */
       private static Path findProjectRoot(Path start) {
           Path cur = start.toAbsolutePath().normalize();
           while (cur != null) {
               if (Files.exists(cur.resolve(".git"))
                       && Files.isRegularFile(cur.resolve("pom.xml"))) {
                   return cur;
               }
               cur = cur.getParent();
           }
           return null;
       }
       ```
