This guarantees A/B comparison fairness: any divergence between Phase 18 and Phase 23 output comes from OSM parsing differences, not from helper drift.

Purpose: additive extraction ŌĆö not a redesign. Phase 18's public behaviour is byte-for-byte unchanged.
Output: new `OsmConversionUtils` class + updated `OsmPipelineService` that delegates to it + a new `OsmConversionUtilsTest` locking in the helper contracts.
