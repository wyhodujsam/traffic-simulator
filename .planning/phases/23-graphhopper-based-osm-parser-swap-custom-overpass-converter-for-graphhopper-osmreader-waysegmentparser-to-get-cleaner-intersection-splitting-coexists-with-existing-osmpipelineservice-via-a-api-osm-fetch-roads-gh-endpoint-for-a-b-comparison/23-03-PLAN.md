<objective>
Implement `GraphHopperOsmService` — the second OSM converter — using GraphHopper 10.2's `WaySegmentParser` (Path B from 23-RESEARCH.md §3). Produces a `MapConfig` for a given bbox that passes `MapValidator`, coexists with Phase 18, and reuses `OsmConversionUtils` for all projection/speed/lane/endpoint/assembly logic (A/B fairness per 23-RESEARCH.md §11).

Per 23-CONTEXT.md locked decision #5, use `com.graphhopper:graphhopper-core:10.2`. Per 23-RESEARCH.md §1, this version is Jackson 2.17.2 lock-step with Spring Boot 3.3.5.

**Scope (from 23-CONTEXT.md decisions 1–9):**
- Service fetches Overpass XML (NOT JSON — per RESEARCH.md §2), writes to a temp file, feeds to `WaySegmentParser`.
- Per-request `BaseGraph` with `RAMDirectory` (per RESEARCH.md §6 thread-safety analysis).
- `EdgeHandler` captures raw `ReaderWay` tags for each segment + node tags for signal detection (per RESEARCH.md §4).
- Output built via `OsmConversionUtils.assembleMapConfig` using `gh-` id prefix.
- Error taxonomy mirrors Phase 18: `IllegalStateException` for 422, `RestClientException` for 503 (per RESEARCH.md §7).
