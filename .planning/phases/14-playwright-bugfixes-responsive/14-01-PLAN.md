  <action>
    The root cause: In `transferVehiclesFromLane` (line 364), when multiple vehicles from the SAME inbound lane are past the threshold, they ALL get transferred to the outbound lane. The gap check on line 388-390 uses `findLeaderAt(outBuffer - 1.0)` which finds the nearest vehicle ahead of outBuffer. After the first vehicle is added at position `outBuffer`, the second vehicle's gap check SHOULD see it — but both are placed at the exact same `outBuffer` position (line 393: `v.updatePhysics(outBuffer, ...)`).

    The fix has two parts:

    1. **Track the last placed position per target lane within the tick.** After transferring a vehicle to a target lane at position P, the next vehicle transferred to the SAME lane must be placed at P + MIN_ENTRY_GAP (not at outBuffer again). Add a `Map<String, Double> lastPlacedPosition` local variable in `processTransfers` (or in `transferVehiclesFromLane`) that tracks the furthest-forward position used per lane ID. Pass this map through.

    2. **In `transferVehiclesFromLane`, update the gap check** (around line 388):
       - After computing `outBuffer`, check `lastPlacedPosition.getOrDefault(targetLane.getId(), -1.0)`.
       - If lastPlacedPosition >= outBuffer, the effective entry position becomes `lastPlacedPosition + MIN_ENTRY_GAP`.
       - Also verify there's room: if effectivePosition > outBuffer + 3 * MIN_ENTRY_GAP, skip (too congested).
       - After successful transfer, update `lastPlacedPosition.put(targetLane.getId(), effectivePosition)`.
       - Use the effectivePosition (not raw outBuffer) in `v.updatePhysics(effectivePosition, ...)`.

    3. **Also fix cross-road same-tick overlap**: The `lastPlacedPosition` map must be shared across ALL calls to `transferVehiclesFromLane` within a single `processTransfers` invocation. Declare it in `processTransfers` and pass it as a parameter to `transferVehiclesFromLane`.

    Update method signatures:
    - `transferVehiclesFromLane(Intersection, String, Lane, RoadNetwork, Map<String, Double>)`
    - In `processTransfers`, create `Map<String, Double> lastPlaced = new HashMap<>()` before the intersection loop and pass it to each `transferVehiclesFromLane` call.

    Do NOT change MIN_ENTRY_GAP value (7.0 is correct: 4.5m vehicle + 2.5m s0).
