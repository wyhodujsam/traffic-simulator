  <action>
    Fix BUG-4: Canvas doesn't scale to available space on large screens.

    The canvas has fixed pixel dimensions from `computeCanvasSize()` (based on road geometry). On Full HD, this results in a small canvas (~630px) with empty space around it.

    Approach: Use CSS `transform: scale()` on the canvas wrapper div to fill the container, maintaining aspect ratio.

    1. **Add a container ref** to measure available space:
       - Add `containerRef = useRef<HTMLDivElement>(null)` for the outer wrapper
       - Use `ResizeObserver` (or `useEffect` + resize listener) to track container dimensions
       - Store `containerWidth` and `containerHeight` in state

    2. **Compute scale factor:**
       ```tsx
       const { width: canvasW, height: canvasH } = computeCanvasSize(roads);
       // containerWidth/Height from ResizeObserver
       const scaleX = containerWidth / canvasW;
       const scaleY = containerHeight / canvasH;
       const scale = Math.min(scaleX, scaleY, 1.5); // cap at 1.5x to avoid blurriness
       // Only scale UP if canvas is smaller than container; never scale down below 1.0
       // Actually: allow scale down too for mobile/small viewports
       const finalScale = Math.max(0.3, Math.min(scale, 2.0));
       ```

    3. **Apply scale to wrapper div:**
       - The wrapper div (line 126) currently has `{ position: 'relative', width, height, margin: 'auto', flexShrink: 0 }`
       - Wrap it in an outer div with `ref={containerRef}` that takes `width: '100%', height: '100%', display: 'flex', alignItems: 'center', justifyContent: 'center', overflow: 'hidden'`
       - On the inner wrapper, add `transform: \`scale(\${finalScale})\``, `transformOrigin: 'center center'`
       - Keep canvas pixel dimensions unchanged (rendering stays crisp)

    4. **Fix click coordinates for scaled canvas:**
       - In `handleCanvasClick`, the `getBoundingClientRect()` already accounts for CSS transforms
       - BUT `cx = e.clientX - rect.left` will be in scaled coordinates
       - Must divide by scale: `cx = (e.clientX - rect.left) / finalScale` (NO — getBoundingClientRect already returns scaled rect)
       - Actually: `rect` from `getBoundingClientRect` reflects the CSS-transformed size. The canvas internal coordinates are unscaled. So: `cx = (e.clientX - rect.left) * (canvas.width / rect.width)` and `cy = (e.clientY - rect.top) * (canvas.height / rect.height)`.
       - Store `finalScale` in a ref so the click handler can access it, OR use the canvas.width/rect.width ratio approach which is self-correcting.

    5. **Handle the container measurement:**
       - Use `ResizeObserver` for robust container size tracking
       - Initialize with a reasonable default (900x600) and update on observe

    Do NOT change `computeCanvasSize` — the internal canvas resolution must stay pixel-accurate for rendering. Only the CSS visual scale changes.
  </action>
