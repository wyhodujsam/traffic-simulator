## Decisions Made

- **Task 3 execution deferred to post-merge orchestrator run** — This worktree is one of 5 parallel agents (Phase 22.1 wave-2). Playwright's `webServer` array binds port 8086 (Spring Boot) and port 5173 (Vite), both process-wide singletons. Running `npx playwright test osm-bbox.spec.ts` in this worktree while sibling agents are also running their own specs would race on those ports and either corrupt each other's state or fail with `EADDRINUSE`. The safe boundary is: in-worktree we author the spec + assert it's discovered by Playwright's test runner (static `--list`); the orchestrator re-runs the merged suite after all wave-2 worktrees land. The user's explicit success criteria for this worktree reflects exactly that split (`Static list check passes: cd frontend && npx playwright test --list e2e/osm-bbox.spec.ts → exit 0`).
- **No tile stub added** — Plan 03 is the reference for tile-stub fallback. Plan 04 mirrors its shape but not its flakiness mitigations; if the spec proves flaky when the orchestrator runs it end-to-end, tile stubbing can be ported from Plan 03 Task 3 as a follow-up.
- **Verbatim plan content preserved, including the comment line `// JSON fixture via fs.readFileSync — avoids deprecated \`assert { type: 'json' }\``** — The plan specifies the spec content verbatim. The plan's automated verify command (`! grep -q "assert { type: 'json' }"`) is stricter than the verbatim content requires (the comment mentions the anti-pattern by name). Resolved in favour of verbatim content; the <done> criterion is about the import form ("NOT via `assert { type: 'json' }`"), which is satisfied. See Deviations §1.
