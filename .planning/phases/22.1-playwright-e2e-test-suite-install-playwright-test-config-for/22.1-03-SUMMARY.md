## Decisions Made

- **JSON fixture import pattern locked to `readFileSync` + `fileURLToPath(new URL(...))`.** Plan CONTEXT explicitly demanded a single committed form for the wave-2 plans to avoid ambiguity. The deprecated `assert { type: 'json' }` ESM attribute and the newer `with { type: 'json' }` variant are NOT used; both are negative-grep verified.
- **Stubbed both vision AND load-config endpoints.** `handleRunSimulation` posts to `/api/command/load-config` before dispatching STOMP START. Without stubbing that REST call, the test would either mutate real backend state (if the real call succeeds) or flip the sidebar to `'error'` state (if it fails). Stubbed with `{ ok: true }` — MapPage only reads `response.ok`, not the body.
- **afterEach POST /api/command { type: 'STOP' } with failOnStatusCode: false.** The STOMP START fired by `sendCommand({ type: 'START' })` is NOT interceptable by `page.route()` (it travels over the WebSocket channel). REST `POST /api/command` dispatches to the same handler and resets the simulation to STOPPED. `failOnStatusCode: false` tolerates cases where the backend is unreachable (parallel-execution mode) so afterEach never fails the test.
- **Run Simulation navigation assert anchored to end-of-path (`/\/$/`)**. `/map` would match `\/$` without anchoring; the explicit end anchor rejects it and only matches the root `/`.
- **Leaflet bbox wait via sidebar dimension text.** Instead of probing internal React state or waiting arbitrary ms, the test waits for `page.getByText(/^\d+m x \d+m$/)` — that element only renders when `bbox !== null` in MapPage state, which only happens after `BoundingBoxMap.onBoundsChange` fires. This is deterministic and selector-based.
- **No Leaflet tile stub added preemptively.** Plan Task 3 described adding `page.route('**/tile.openstreetmap.org/**', ...)` only IF the bbox-wait times out in the real run. Since Task 3's full run is deferred (see Deviations), no tile stub has been added — the spec matches the "baseline" form. If Plan 22.1-06 wave discovers tile-load blocking the bbox wait, they add it then.
