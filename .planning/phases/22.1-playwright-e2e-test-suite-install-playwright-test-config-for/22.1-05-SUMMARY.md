## Decisions Made

- **Per-describe viewport pattern** — `test.use({ viewport: ... })` inside each `test.describe` block. Chosen over per-test `page.setViewportSize()` for clarity (each describe owns one viewport) and over project-level configs (no need to add two Playwright projects for a single spec).
- **Semantic locators over data-testid** — `<main>` and `<aside>` are the natural ARIA roles for the two flex children on SimulationPage; only one of each exists on the page (DisconnectBanner uses `<div>`). No production changes required to support the spec.
- **Static `--list` as wave-2 gate** — Full runtime execution (spawning Spring Boot + Vite webServer) is incompatible with parallel-wave execution: port conflicts with other agents, 120s backend cold-start, backend not required for layout geometry assertions. The plan's runtime step is logically a wave-3 integration verification.
- **50vh canvas height cap asserted directly** — `mainBox.height <= 667 * 0.5 + 4` is a direct structural assertion of the `maxHeight: isMobile ? '50vh' : undefined` rule in SimulationPage.tsx. This catches regressions if the mobile cap is removed.
- **Tolerance values** — 2px on adjacency checks (subpixel rendering), 4px on width/height caps (border+padding), 16px on mobile sidebar width (scrollbar+padding), 100px on desktop vertical alignment (DisconnectBanner shift). Strict equality would be flaky.
