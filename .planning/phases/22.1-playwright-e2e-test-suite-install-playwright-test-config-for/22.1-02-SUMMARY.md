## Decisions Made

- **Spec content verbatim from plan's task 1 `<action>` block.** The plan provided the full 92-line spec inline; no editorial changes were needed. Plan's embedded comments already justified every design choice (WebSocket wait, async-option gate, poll intervals, trailing-spawn tolerance).
- **Static `--list` verification substituted for full spec run.** User-level success criterion explicitly states "Test is NOT expected to run in the worktree (backend not available) — verify by a static syntax check: `npx playwright test --list e2e/simulation.spec.ts` lists exactly one test case." Attempting `npx playwright test simulation.spec.ts` would have invoked the webServer which cold-starts `mvn spring-boot:run` (Maven may not be accessible from this worktree path) and would have produced a misleading failure unrelated to the spec's correctness. The static listing exercises Playwright's TypeScript parser + config loader + test discovery, which is sufficient to prove the spec is syntactically valid and findable.
- **Hardcoded `http://localhost:8086/api/command` in afterEach** (not a relative URL) — `request.post` in afterEach does not inherit the `page` baseURL, so a relative path would fail. Matches the plan's verbatim source.
