## Decisions Made

- **Used `--legacy-peer-deps` for install** — The pre-existing peer conflict between `vitest@1.6.1` and `@vitest/coverage-v8@4.1.4` blocked a clean `npm install`. Rule 3 (blocking issue) auto-fix. Does not affect runtime correctness; the conflict predates this plan and is out of scope for Phase 22.1.
- **Chromium install without `--with-deps`** — `npx playwright install --with-deps chromium` requires `sudo apt install` for system libraries; sandbox lacks interactive sudo. Fell back to `npx playwright install chromium` per plan's explicit fallback clause. Host already has the required shared libs from prior browser installs; Playwright confirmed functional via `npx playwright --version` (1.59.1).
- **Added `--pass-with-no-tests` to `test:e2e` script** — Playwright 1.59 exits non-zero from `playwright test --list` when zero spec files are present, even when the config parses cleanly. The plan's must-have truth (`npm run test:e2e -- --list` exits 0 with zero specs) is satisfied by baking this flag into the script. Wave-2 plans (22.1-02…06) will add specs but the smoke-check semantics remain consistent.
