# Plan 4.1: Wire PhysicsEngine into TickEmitter with Speed Multiplier

## Goal

Inject `PhysicsEngine` into `TickEmitter` and call it for every lane in every road each tick. Apply `speedMultiplier` to `dt` for both physics and spawner. Add sub-stepping for high multipliers to maintain IDM stability.

## Must-haves

- PhysicsEngine.tick(lane, dt) called for every lane every tick (between spawn and despawn)
- Speed multiplier scales effective dt
- Sub-stepping when effectiveDt > 0.05s (prevents Euler instability at high speed)
- Existing 38 tests still pass

Replace with:

```java
if (simulationEngine.getStatus() == SimulationStatus.RUNNING) {
    tick = simulationEngine.getTickCounter().incrementAndGet();

    // Run simulation pipeline when network is loaded
    if (network != null) {
        double baseDt = 0.05; // 50ms = 1/20 Hz
        double multiplier = simulationEngine.getSpeedMultiplier();
        double effectiveDt = baseDt * multiplier;

        // Sub-stepping: keep each physics step <= baseDt for Euler stability
        int subSteps = Math.max(1, (int) Math.ceil(effectiveDt / baseDt));
        double stepDt = effectiveDt / subSteps;

        // 1. Spawn (uses full effectiveDt for accumulator)
        vehicleSpawner.tick(effectiveDt, network, tick);

        // 2. Physics sub-steps
        for (int step = 0; step < subSteps; step++) {
            for (Road road : network.getRoads().values()) {
                for (Lane lane : road.getLanes()) {
                    physicsEngine.tick(lane, stepDt);
                }
            }
        }

        // 3. Despawn (after physics moved vehicles)
        vehicleSpawner.despawnVehicles(network);
    }
}
```

Also add these imports at the top of the file (if not already present):
```java
import com.trafficsimulator.model.Road;
import com.trafficsimulator.model.Lane;
```

Note: `Road` and `Lane` are already imported. No new imports needed beyond `PhysicsEngine` from Task 4.1.1.
