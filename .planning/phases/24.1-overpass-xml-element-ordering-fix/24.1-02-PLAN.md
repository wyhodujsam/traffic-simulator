<objective>
Close the critical test-coverage gap that allowed the Phase 23/24 Overpass XML ordering bug to ship: there is currently NO test in the default `mvn test` run that exercises the real `Osm2StreetsService.fetchAndConvert(bbox)` or `GraphHopperOsmService.fetchAndConvert(bbox)` end-to-end against a realistic Overpass payload, AND no test that asserts on the Overpass query SHAPE actually sent over HTTP.

This plan adds a single `@SpringBootTest`-based integration test, `OsmPipelineSmokeIT`, that:
1. Loads the full Spring context (real `OverpassXmlFetcher`, real `Osm2StreetsService`, real `GraphHopperOsmService`, real `MapValidator`, real `Osm2StreetsConfig`).
2. Binds `MockRestServiceServer` to the application's `overpassRestClient` bean so the Overpass HTTP request is intercepted INSIDE the JVM and answered with a canned XML payload from disk — no external network, no flakiness.
3. Calls both services' `fetchAndConvert(bbox)`, exercising the real subprocess (osm2streets-cli) and the real WaySegmentParser (GraphHopper) on the canned XML.
4. Asserts both produce ≥ 1 road and pass `MapValidator`.
5. **CRITICAL — addresses checker Issue #1:** captures the URL-encoded Overpass POST body sent by each service via a custom `RequestMatcher` and asserts on the produced query shape: positive contract (`>;);out body qt;` whitespace-insensitive — the recursion-in-union form delivered by Plan 01) and negative contract (no `out body;` followed by `>;` — the buggy two-pass form). This makes Plan 02 a true regression net: reverting Plan 01's fix to `out body;\n>;\nout body qt;` would now FAIL these assertions, even though the canned XML alone cannot detect that revert.

Purpose: future regressions to either the Overpass query string OR the XML parsing path OR the subprocess wiring will fail this test in plain `mvn test` — the disabled-by-default `OsmPipelineComparisonTest.compareAllConverters_sameBbox_logDiff` no longer carries the safety net alone, AND Plan 01's unit test contract is now coupled to Plan 02's integration test by the request-body capture.

Output:
- `backend/src/test/resources/osm/overpass-real-modlin.xml` — frozen Overpass XML fixture with nodes-before-ways ordering, generated from the FIXED query (Plan 01 must be in master before this plan runs).
- `backend/src/test/java/com/trafficsimulator/service/OsmPipelineSmokeIT.java` — `@SpringBootTest`, runs by default, captures + asserts on request bodies.
</objective>

**Custom RequestMatcher for body capture (the Issue #1 fix):**

`MockRestServiceServer` accepts custom `org.springframework.test.web.client.RequestMatcher` lambdas. The body is available via `MockClientHttpRequest.getBodyAsString()`. A capturing matcher looks like:
```java
RequestMatcher captureBody = (request) -> {
    MockClientHttpRequest mock = (MockClientHttpRequest) request;
    String body = mock.getBodyAsString();   // throws IOException; declare in lambda via runtime wrap
    capturedRequestBodies.add(body);          // stash for post-call assertions
};
```
The captured body is the URL-encoded form `data=%5Bout%3Axml%5D...` — decode with `URLDecoder.decode(body.substring(5), UTF_8)` (strip the `data=` prefix), then assert the decoded form contains `>;);out body qt;` (whitespace-insensitive) AND does NOT contain `out body;` followed by `>;`.

Note: `MockRestServiceServer.bindTo` accepts a `RestClient.Builder` OR a built `RestClient` (Spring 6+). Verify which the codebase exposes — the `OverpassXmlFetcher` constructor takes a `RestClient` (already built), and the underlying `ClientHttpRequestFactory` is what gets stubbed. If the `RestClient` was built in such a way that prevents binding (e.g. fixed factory), use `MockRestServiceServer.bindTo(restClient)` directly; if that does not work, the alternative is to register a test `@Configuration` that exposes a `RestClient.Builder` whose `RestTemplate` adapter can be intercepted. **Action plan: try `MockRestServiceServer.bindTo(overpassRestClient)` first; fall back to registering a `@TestConfiguration` exposing a builder if it does not compile/work.**

     *   <li>NEGATIVE: does NOT contain {@code out body;} followed by {@code >;} (the buggy

    @TestPropertySource(properties = "osm.overpass.urls=http://test-overpass.local")
    @EnabledOnOs(OS.LINUX)

        /**
         * Modlin / Nowy Dwór Mazowiecki bbox — the same area documented in 24.1-CONTEXT.md as the
         * real-world reproducer for the bug.
         */
        private static final BboxRequest BBOX =
                new BboxRequest(52.431, 20.65, 52.438, 20.662);

        /**
         * Positive query-shape contract: recursion `>;` lives INSIDE the union (immediately before
         * the closing `);`) and the trailing directive is `out body qt;`. Whitespace-insensitive so
         * formatting changes in {@link OverpassXmlFetcher#buildOverpassXmlQuery} do not break this.
         */
        private static final Pattern POSITIVE_QUERY_SHAPE =
                Pattern.compile(">;\\s*\\);\\s*out\\s+body\\s+qt;");

        /**
         * Negative query-shape contract: the buggy two-pass form `out body;\n>;` MUST NOT appear.
         * This is the EXACT regression-net pattern: reverting Plan 01's fix would re-introduce
         * this substring and trip this assertion.
         */
        private static final Pattern NEGATIVE_QUERY_SHAPE =
                Pattern.compile("out\\s+body;\\s*\\n\\s*>;");

        @Autowired private RestClient overpassRestClient;
        @Autowired private Osm2StreetsService osm2StreetsService;
        @Autowired private GraphHopperOsmService graphHopperOsmService;
        @Autowired private MapValidator mapValidator;

        private MockRestServiceServer mockServer;
        private String cannedOverpassXml;
        private final List<String> capturedQueries = new ArrayList<>();

        /**
         * Captures the URL-decoded Overpass query string from each POST. The body shape is
         * `data=%5Bout%3Axml%5D...` — the prefix `data=` is stripped and the rest URL-decoded.
         */
        private final RequestMatcher captureBody = request -> {
            MockClientHttpRequest mock = (MockClientHttpRequest) request;
            String rawBody = mock.getBodyAsString();
            String formValue = rawBody.startsWith("data=") ? rawBody.substring("data=".length()) : rawBody;
            String decoded = URLDecoder.decode(formValue, StandardCharsets.UTF_8);
            capturedQueries.add(decoded);
        };

            capturedQueries.clear();

            Path fixture = Paths.get(
                    Objects.requireNonNull(

                                            .getClassLoader()
                                            .getResource("osm/overpass-real-modlin.xml"),
                                    "fixture not found: osm/overpass-real-modlin.xml")

        /**
         * Asserts the captured Overpass query body has the corrected shape. Called from each
         * service's @Test method AND from the dedicated regression-guard test.
         */
        private void assertOverpassQueryShape(String capturedBody, String contextLabel) {
            assertThat(capturedBody)
                    .as("[%s] captured Overpass query must use recursion-in-union form (`>;);out body qt;`)", contextLabel)
                    .containsPattern(POSITIVE_QUERY_SHAPE);
            assertThat(capturedBody)
                    .as("[%s] captured Overpass query MUST NOT use the buggy two-pass form (`out body;\\n>;`)", contextLabel)
                    .doesNotContainPattern(NEGATIVE_QUERY_SHAPE);
            assertThat(capturedBody)
                    .as("[%s] captured query must include the fixed bbox in S,W,N,E order", contextLabel)
                    .contains("52.431", "20.65", "52.438", "20.662");
        }

        @Test
        void osm2StreetsService_fetchAndConvert_realBinary_returnsAtLeastOneRoad() {
            mockServer
                    .expect(requestTo(CoreMatchers.endsWith("/api/interpreter")))
                    .andExpect(method(HttpMethod.POST))
                    .andExpect(captureBody)
                    .andRespond(withSuccess(cannedOverpassXml, MediaType.APPLICATION_XML));

            MapConfig cfg = osm2StreetsService.fetchAndConvert(BBOX);

            assertThat(cfg).as("osm2streets must return a MapConfig").isNotNull();
            assertThat(cfg.getRoads())
                    .as("osm2streets must return ≥ 1 road for the canned Modlin XML")
                    .isNotNull()
                    .isNotEmpty();
            assertThat(mapValidator.validate(cfg))
                    .as("osm2streets output must pass MapValidator")
                    .isEmpty();

                    .allSatisfy(r -> assertThat(r.getId()).startsWith("o2s-"));

            // Issue #1: the query the service ACTUALLY sent must be the corrected shape.
            assertThat(capturedQueries).as("exactly one Overpass POST captured").hasSize(1);
            assertOverpassQueryShape(capturedQueries.get(0), "Osm2StreetsService");

            mockServer.verify();
        }

        @Test
        void graphHopperOsmService_fetchAndConvert_realParser_returnsAtLeastOneRoadAndOneSignalIntersection() {
            mockServer
                    .expect(requestTo(CoreMatchers.endsWith("/api/interpreter")))
                    .andExpect(method(HttpMethod.POST))
                    .andExpect(captureBody)
                    .andRespond(withSuccess(cannedOverpassXml, MediaType.APPLICATION_XML));

            assertThat(cfg).as("GraphHopper must return a MapConfig").isNotNull();
            assertThat(cfg.getRoads())
                    .as("GraphHopper must return ≥ 1 road for the canned Modlin XML")
                    .isNotNull()
                    .isNotEmpty();

            assertThat(cfg.getIntersections())
                    .as("GraphHopper must detect the traffic_signals node as a SIGNAL intersection")
                    .anySatisfy(ic -> assertThat(ic.getType()).isEqualTo("SIGNAL"));

            assertThat(mapValidator.validate(cfg))
                    .as("GraphHopper output must pass MapValidator")
                    .isEmpty();

            assertThat(cfg.getRoads())
                    .allSatisfy(r -> assertThat(r.getId()).startsWith("gh-"));

            // Issue #1: the query the service ACTUALLY sent must be the corrected shape.
            assertThat(capturedQueries).as("exactly one Overpass POST captured").hasSize(1);
            assertOverpassQueryShape(capturedQueries.get(0), "GraphHopperOsmService");

            mockServer.verify();
        }

        /**
         * Phase 24.1 Issue #1 regression net: explicitly couples the produced Overpass query
         * shape (Plan 01's contract) to the integration test (Plan 02's harness). Reverting
         * Plan 01's fix to `out body;\n>;\nout body qt;` causes this test to FAIL — making Plan 02
         * a true backstop for the original bug class even though the canned XML alone cannot
         * detect such a revert.
         *

         * coverage of both the {@code fetchXmlBytes} (Osm2StreetsService) and
         * {@code fetchXmlToTempFile} (GraphHopperOsmService) code paths in a single test.
         */
        @Test
        void bothServicesEmitRecursionInsideUnionAndSingleOutBodyQt_regressionGuard() {
            // Expect TWO POSTs (one per service) — both captured.
            mockServer
                    .expect(requestTo(CoreMatchers.endsWith("/api/interpreter")))
                    .andExpect(method(HttpMethod.POST))
                    .andExpect(captureBody)
                    .andRespond(withSuccess(cannedOverpassXml, MediaType.APPLICATION_XML));

                    .expect(requestTo(CoreMatchers.endsWith("/api/interpreter")))
                    .andExpect(method(HttpMethod.POST))
                    .andExpect(captureBody)
                    .andRespond(withSuccess(cannedOverpassXml, MediaType.APPLICATION_XML));

            // Drive both services — discard the MapConfig outputs; this test only asserts on the
            // captured query shapes.

                    .as("two captured Overpass POSTs (one per service)")

            assertThat(capturedQueries.get(0))

            mockServer.verify();
        }
