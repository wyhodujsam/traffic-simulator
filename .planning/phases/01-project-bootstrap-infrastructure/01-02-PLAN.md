## Task 1.2.2: Configure Vite dev server proxy

<read_first>
- /home/sebastian/traffic-simulator/frontend/vite.config.ts
- /home/sebastian/traffic-simulator/.planning/phases/01-project-bootstrap-infrastructure/01-RESEARCH.md (section 2, Vite proxy config)
</read_first>

<action>
Replace `frontend/vite.config.ts` with:

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 5173,
    proxy: {
      '/ws': {
        target: 'http://localhost:8080',
        ws: true,
        changeOrigin: true,
      },
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
      },
    },
  },
})
```

The `ws: true` flag is critical for WebSocket upgrade handshake proxying. Without it, SockJS falls back to long-polling.
</action>
