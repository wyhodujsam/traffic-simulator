Prompt constant (from RESEARCH §"Prompt Design", expanded):

```
You are a traffic-map recogniser. Look at the road image and identify which
PREDEFINED COMPONENTS appear, where they are, and how they connect.

VALID COMPONENT TYPES (you MUST NOT invent new types — if you cannot fit a
region into one of these, OMIT IT):

- ROUNDABOUT_4ARM — circular ring with up to 4 arms (north/east/south/west)
  fields: id (lowercase alnum, no 'in'/'out'), centerPx{x,y}, rotationDeg,
          armsPresent (subset of [north,east,south,west])
- SIGNAL_4WAY — signalised intersection with up to 4 approaches
  fields: id, centerPx{x,y}, rotationDeg, armsPresent
- T_INTERSECTION — 3 arms, PRIORITY control
  fields: id, centerPx{x,y}, rotationDeg (0 = stem points south),
          armsPresent (exactly 3 of [north,east,south,west])
- STRAIGHT_SEGMENT — connector road between two component arms
  fields: id, startPx{x,y}, endPx{x,y}, lengthPx

OUTPUT FORMAT (raw JSON only, no markdown fences, no prose):
{
  "components": [ { "type": "...", "id": "...", ... }, ... ],
  "connections": [ { "a": "<componentId>.<armName>",
                     "b": "<componentId>.<armName>" }, ... ]
}

RULES:
- Component ids MUST match ^[a-z][a-z0-9]*$ and MUST NOT contain 'in' or 'out'.
  Safe prefixes: rb, sig, t, seg, then a digit (rb1, sig2, t3, seg4).
- If two component arms meet at the same pixel location, emit one connection.
- If two arms are separated by visible road, insert a STRAIGHT_SEGMENT and emit
  two connections (one per end).
- Unconnected arms = network boundaries (traffic enters/exits there). Fine.
- At most ONE non-connector component per pixel location.
```

Two-pass DTO per RESEARCH §"Code Examples":

```java
public class ComponentSpecDto {
    public String type;
    public String id;
    public Point2D.Double centerPx;
    public Point2D.Double startPx;
    public Point2D.Double endPx;
    public double rotationDeg;
    public java.util.List<String> armsPresent;
    public Double lengthPx;
}
public class ComponentEnvelopeDto {
    public List<ComponentSpecDto> components;
    public List<ConnectionDto> connections;
}
public class ConnectionDto { public String a; public String b; }
```

Mapping switch:
```java
ComponentSpec toSpec(ComponentSpecDto d) {
    return switch (d.type) {
        case "ROUNDABOUT_4ARM" -> new RoundaboutFourArm(d.id, d.centerPx, d.rotationDeg, d.armsPresent);
        case "SIGNAL_4WAY"     -> new SignalFourWay(d.id, d.centerPx, d.rotationDeg, d.armsPresent);
        case "T_INTERSECTION"  -> new TIntersection(d.id, d.centerPx, d.rotationDeg, d.armsPresent);
        case "STRAIGHT_SEGMENT"-> new StraightSegment(d.id, d.startPx, d.endPx, d.lengthPx == null ? 0 : d.lengthPx);
        default -> throw new ClaudeCliParseException("Unknown component type: " + d.type
            + ". Valid types: ROUNDABOUT_4ARM, SIGNAL_4WAY, T_INTERSECTION, STRAIGHT_SEGMENT");
    };
}
```
