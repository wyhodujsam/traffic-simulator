<!-- The wrapper is a tiny binary that reads OSM XML bytes on stdin, -->
<!-- produces StreetNetwork JSON on stdout, exit 0 on success. -->

The wrapper MUST accept EXACTLY this protocol (Plan 24-03 contracts against it):
  - stdin:  OSM XML bytes (UTF-8)
  - stdout: serde-serialized StreetNetwork JSON (UTF-8, newline-terminated OK)
  - stderr: human-readable log lines (ignored by Java unless exit != 0)
  - exit 0: success; stdout is valid JSON
  - exit !=0: failure; stderr carries the reason

Target triple: x86_64-unknown-linux-musl (static, ~15 MB)
</interfaces>
