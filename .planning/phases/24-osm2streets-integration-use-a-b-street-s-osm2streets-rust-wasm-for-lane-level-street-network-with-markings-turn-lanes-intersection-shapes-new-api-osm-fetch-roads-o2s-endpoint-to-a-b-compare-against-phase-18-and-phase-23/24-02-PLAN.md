Extend `MapConfig.RoadConfig` with an OPTIONAL `lanes: List<LaneConfig>` field and a new nested `LaneConfig` record so Plan 24-04's mapper can surface osm2streets' per-lane metadata. MUST be additive and byte-identical-on-wire for existing Phase 18 and Phase 23 producers.

Output: Extended MapConfig.java, one new MapValidatorTest assertion (populated lanes → 0 validation errors), and regression assertions in Phase 18/23 test classes that `lanes` stays null for their output.
</objective>
