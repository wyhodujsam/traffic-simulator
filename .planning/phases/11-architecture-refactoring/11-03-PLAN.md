# Plan 11-03: Extract SnapshotBuilder from TickEmitter

## Goal

Extract the ~130-line snapshot-building logic from TickEmitter into a dedicated `SnapshotBuilder` component. TickEmitter becomes a pure orchestrator (~80 lines). This separates scheduling concerns from DTO projection.

## Context

TickEmitter currently has two responsibilities: (1) orchestrate the tick pipeline (drain commands, spawn, physics, lane changes, despawn) and (2) build the SimulationStateDto snapshot with pixel projection, statistics, and traffic light DTOs. The second responsibility is ~130 lines that belong in a dedicated class.

After Plan 11-01, Lane exposes `getVehiclesView()` / `getObstaclesView()` — SnapshotBuilder must use these.
