# Plan 11-02: Engine Interfaces and Dependency Inversion

## Goal

Extract interfaces for all 5 engine components. TickEmitter and LaneChangeEngine depend on interfaces instead of concrete classes. This enables testability via mocking and decouples the tick pipeline from implementations.

## Context

Currently TickEmitter depends on concrete PhysicsEngine, LaneChangeEngine, TrafficLightController, IntersectionManager, VehicleSpawner. LaneChangeEngine depends on concrete PhysicsEngine. No interfaces exist. This prevents unit testing TickEmitter in isolation and violates Dependency Inversion Principle.

---
