# Plan 11-01: Encapsulate Collections in Lane and RoadNetwork

## Goal

Make `vehicles` and `obstacles` lists in Lane private, exposing only controlled mutation methods. Add query methods to RoadNetwork. This eliminates 5+ classes directly mutating internal collections and enforces encapsulation.

## Context

Currently `lane.getVehicles()` returns a mutable `List<Vehicle>` and 7 classes call `.add()`, `.remove()`, `.removeIf()`, `.clear()`, `.size()` directly on it. Same for `lane.getObstacles()`. This breaks encapsulation and makes it impossible to maintain sorted invariants or add validation later.

---

## Tasks

### Task 1: Add collection mutation methods to Lane

<read_first>
- `backend/src/main/java/com/trafficsimulator/model/Lane.java`
</read_first>

<action>
Add the following methods to Lane.java and make `vehicles` and `obstacles` fields encapsulated:

1. Remove `@Data` (which generates public getters for List fields) — replace with explicit getters
2. Keep `@Builder` but add `@Getter` only for non-collection fields (id, laneIndex, road, length, maxSpeed, active)
3. Add methods for vehicle management:
   - `addVehicle(Vehicle v)` — adds vehicle to internal list
   - `removeVehicle(Vehicle v)` — removes specific vehicle
   - `removeVehiclesIf(Predicate<Vehicle> predicate)` — replaces `getVehicles().removeIf()`
   - `getVehicleCount()` — replaces `getVehicles().size()`
   - `getVehiclesView()` — returns `Collections.unmodifiableList(vehicles)`
   - `clearVehicles()` — replaces `getVehicles().clear()`
4. Add methods for obstacle management:
   - `addObstacle(Obstacle o)` — adds obstacle to internal list
   - `removeObstacle(Obstacle o)` — removes specific obstacle
   - `removeObstaclesIf(Predicate<Obstacle> predicate)` — replaces `getObstacles().removeIf()`
   - `getObstaclesView()` — returns `Collections.unmodifiableList(obstacles)`
   - `clearObstacles()` — replaces `getObstacles().clear()`

Keep the existing `getVehicles()` temporarily as a **deprecated** method returning the mutable list to enable incremental migration, then remove it in the final task.
</action>

### Task 2: Add query methods to RoadNetwork

<read_first>
- `backend/src/main/java/com/trafficsimulator/model/RoadNetwork.java`
- `backend/src/main/java/com/trafficsimulator/model/Road.java`
</read_first>

<action>
Add convenience query methods to RoadNetwork:

```java
/** Flat stream of all lanes across all roads */
public Stream<Lane> getAllLanes() {
    return roads.values().stream()
        .flatMap(road -> road.getLanes().stream());
}

/** Flat stream of all vehicles across all lanes */
public Stream<Vehicle> getAllVehicles() {
    return getAllLanes()
        .flatMap(lane -> lane.getVehiclesView().stream());
}

/** Find road by ID */
public Optional<Road> findRoad(String id) {
    return Optional.ofNullable(roads.get(id));
}

/** Find lane by global lane ID (e.g. "r1-lane0") */
public Optional<Lane> findLane(String laneId) {
    return getAllLanes()
        .filter(lane -> lane.getId().equals(laneId))
        .findFirst();
}
```
</action>
