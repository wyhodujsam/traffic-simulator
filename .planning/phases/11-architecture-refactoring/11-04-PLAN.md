# Plan 11-04: Extract CommandDispatcher from SimulationEngine

## Goal

Extract the 150-line `applyCommand()` switch from SimulationEngine into a dedicated CommandDispatcher component. Implement the missing LoadMap handler (bug C2 from review). SimulationEngine becomes a pure state container + queue manager (~100 lines).

## Context

SimulationEngine currently has two responsibilities: (1) state container (status, roadNetwork, counters, rates) with enqueue/drain and (2) command processing (150-line if-else chain in `applyCommand()`). The command handler also has a stub LoadMap that only logs — it needs full implementation.

**handleLoadMap implementation (was missing - bug C2):**
```java
private void handleLoadMap(SimulationCommand.LoadMap cmd) {
    if (mapLoader == null) {
        log.warn("MapLoader not available — ignoring LoadMap command");
        return;
    }
    // Stop simulation first
    engine.setStatus(SimulationStatus.STOPPED);
    engine.getTickCounter().set(0);
    if (vehicleSpawner != null) vehicleSpawner.reset();

    try {
        RoadNetwork network = mapLoader.loadFromClasspath("maps/" + cmd.mapId() + ".json");
        engine.setRoadNetwork(network);
        log.info("Map loaded: {}", cmd.mapId());
    } catch (Exception e) {
        log.error("Failed to load map {}: {}", cmd.mapId(), e.getMessage());
    }
}
```
