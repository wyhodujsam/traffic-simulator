# Plan 09-03: Frontend Map Selector & Road Re-fetch

## Goal

Add a map selector dropdown to ControlsPanel that fetches available maps from `GET /api/maps`, sends `LOAD_MAP` command on selection, re-fetches road geometry after map load, and displays any map loading errors. The dropdown is disabled while the simulation is RUNNING.

### Task 4: Add map selector dropdown to ControlsPanel

<read_first>
- `frontend/src/components/ControlsPanel.tsx` (current layout and styling)
- `frontend/src/store/useSimulationStore.ts` (store selectors)
</read_first>

<action>
Add map selector dropdown to `ControlsPanel.tsx` above the simulation buttons:

1. **Subscribe to store values:**
```typescript
const availableMaps = useSimulationStore((s) => s.availableMaps);
const currentMapId = useSimulationStore((s) => s.currentMapId);
const mapError = useSimulationStore((s) => s.mapError);
const refetchRoads = useSimulationStore((s) => s.refetchRoads);
```

2. **Map selection handler:**
```typescript
const handleMapChange = (e: React.ChangeEvent<HTMLSelectElement>) => {
  const mapId = e.target.value;
  if (!mapId || mapId === currentMapId) return;
  send({ type: 'LOAD_MAP', mapId });
  // Re-fetch roads after short delay to allow backend to load new map
  setTimeout(() => {
    if (refetchRoads) refetchRoads();
  }, 200);
};
```

3. **Render dropdown** — place before simulation buttons, disabled when RUNNING:
```tsx
{/* Map selector */}
<div style={{ marginBottom: '16px' }}>
  <label style={{ display: 'block', marginBottom: '4px' }}>
    Scenario:
  </label>
  <select
    value={currentMapId ?? ''}
    onChange={handleMapChange}
    disabled={status === 'RUNNING'}
    style={{
      width: '100%',
      padding: '6px 8px',
      background: '#2a2a3e',
      color: '#e0e0e0',
      border: '1px solid #555',
      borderRadius: '4px',
      fontFamily: 'monospace',
      fontSize: '13px',
      cursor: status === 'RUNNING' ? 'not-allowed' : 'pointer',
    }}
  >
    {availableMaps.map((m) => (
      <option key={m.id} value={m.id}>
        {m.name}
      </option>
    ))}
  </select>
  {/* Show description of selected map */}
  {availableMaps.find(m => m.id === currentMapId)?.description && (
    <div style={{ fontSize: '11px', color: '#888', marginTop: '4px' }}>
      {availableMaps.find(m => m.id === currentMapId)?.description}
    </div>
  )}
</div>
```

4. **Error banner** — show mapError if set:
```tsx
{mapError && (
  <div style={{
    background: '#3a1a1a',
    border: '1px solid #ff4444',
    borderRadius: '4px',
    padding: '8px',
    marginBottom: '12px',
    fontSize: '12px',
    color: '#ff6b6b',
  }}>
    {mapError}
  </div>
)}
```

Place error banner right after the map selector div.
</action>

<acceptance_criteria>
- Dropdown shows all available maps with names
- Selecting a map sends LOAD_MAP command via STOMP
- Roads are re-fetched 200ms after map selection
- Dropdown is disabled when status is RUNNING

<action>
Ensure canvas redraws when roads change after map load:

1. In the map change handler in ControlsPanel, before re-fetching roads, clear stale state:
```typescript
const handleMapChange = (e: React.ChangeEvent<HTMLSelectElement>) => {
  const mapId = e.target.value;
  if (!mapId || mapId === currentMapId) return;

  // Clear stale snapshots to prevent old vehicles rendering on new map
  useSimulationStore.getState().clearSnapshots();

  send({ type: 'LOAD_MAP', mapId });
  setTimeout(() => {
    if (refetchRoads) refetchRoads();
  }, 200);
};
```

2. Add `clearSnapshots` action to store:
```typescript
clearSnapshots: () => set({
  currSnapshot: null,
  prevSnapshot: null,
  obstacles: [],
  trafficLights: [],
}),
```

This ensures no stale vehicles from the old map render briefly on the new map geometry.
</action>

<acceptance_criteria>
- Snapshots are cleared when user selects a new map
- No stale vehicles render on the new map
- Canvas redraws with new road geometry after re-fetch
- `roadsLoaded` stays true between map changes (setRoads always sets it to true)
</acceptance_criteria>
