  <action>
The root cause: App.tsx line 79-87 uses `alignItems: 'center'` and `justifyContent: 'center'` on the `<main>` flex container. When the canvas (2100px wide) overflows the `<main>`, flex centering pushes the content into "negative scroll" territory — the browser cannot scroll to the left/top portion of the canvas.

**Fix in App.tsx (line 79-87):**
Remove `alignItems: 'center'` and `justifyContent: 'center'` from the `<main>` style. The `<main>` style should become:
```
{
  flex: 1,
  minWidth: 0,
  display: 'flex',
  padding: '16px',
  overflow: 'auto',
  position: 'relative',
}
```
(No alignItems, no justifyContent — centering is handled by margin: auto on the child.)

**Fix in SimulationCanvas.tsx (line 126):**
Add `margin: 'auto'` to the wrapper `<div>` style (currently `{ position: 'relative', width, height }`). Change to:
```
{ position: 'relative', width, height, margin: 'auto', flexShrink: 0 }
```
- `margin: 'auto'` centers the canvas when it's smaller than the viewport
- `flexShrink: 0` prevents the canvas from being compressed (it should scroll instead)

This gives the best of both worlds: centered when small, scrollable top-left-anchored when large.
