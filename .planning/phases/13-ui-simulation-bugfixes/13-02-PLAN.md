```java
String signalState = tl.getSignalState(inRoadId);
if ("GREEN".equals(signalState)) {
    // Check if all outbound roads are full (box-blocking active)
    boolean anyOutboundHasSpace = false;
    Intersection ixtn2 = ixtn; // already in scope
    for (String outRoadId : ixtn2.getOutboundRoadIds()) {

        if (outRoadId.replace("_in", "_out").equals(inRoadId.replace("_in", "_out"))) continue;
        Road outRoad = network.getRoads().get(outRoadId);
        if (outRoad == null) continue;
```

**Two visual problems to fix:**

**Problem A — Light position/angle confusion:**
Currently, the traffic light is drawn at `(tl.x, tl.y)` which is `inRoad.endX/endY` — the road endpoint at intersection center. For a 4-way intersection, all 4 roads converge near the center, so lights overlap or are indistinguishable.

Fix: Offset the traffic light BACKWARD along the road direction (away from intersection) so it sits at the stop line, not at the center. Add a perpendicular offset so the light sits to the side of the road, not on top of it.

Change the drawing code to:
```typescript
const STOP_LINE_OFFSET = 25; // pixels back from intersection
const ROAD_SIDE_OFFSET = LANE_WIDTH_PX * 1.2; // offset to the right side of road

for (const tl of trafficLights) {
  ctx.save();
  
  // Offset backward along road direction to stop line position
  const cos = Math.cos(tl.angle);
  const sin = Math.sin(tl.angle);
  const stopX = tl.x - cos * STOP_LINE_OFFSET;
  const stopY = tl.y - sin * STOP_LINE_OFFSET;
  
  // Offset perpendicular (to the right of the road direction)
  const perpX = -sin * ROAD_SIDE_OFFSET;
  const perpY = cos * ROAD_SIDE_OFFSET;
  
  ctx.translate(stopX + perpX, stopY + perpY);
  ctx.rotate(tl.angle);
  
  // Draw background box for the 3 lights
  const boxW = LIGHT_SPACING * 2 + LIGHT_RADIUS * 2 + 6;
  const boxH = LIGHT_RADIUS * 2 + 6;
  ctx.fillStyle = '#1a1a1a';
  ctx.fillRect(-boxW / 2, -boxH / 2, boxW, boxH);
  
  // Draw three circles: RED, YELLOW, GREEN (left to right)
  const states: Array<'RED' | 'YELLOW' | 'GREEN'> = ['RED', 'YELLOW', 'GREEN'];
  for (let i = 0; i < states.length; i++) {
    const s = states[i];
    const isActive = tl.state === s;
    ctx.beginPath();
    ctx.arc(-LIGHT_SPACING + i * LIGHT_SPACING, 0, LIGHT_RADIUS, 0, Math.PI * 2);
    ctx.fillStyle = isActive ? COLORS[s].active : COLORS[s].dim;
    ctx.fill();
  }
```

**Problem B — Box-blocking visual cue:**
When `tl.boxBlocked` is true and state is GREEN, draw a small "BLOCKED" indicator — an orange outline around the traffic light box and a small "X" symbol.

After drawing the 3 light circles, add:

```typescript
  // Box-blocking indicator: orange border when green but blocked
  if (tl.boxBlocked && tl.state === 'GREEN') {
    ctx.strokeStyle = '#ff8800';

    ctx.strokeRect(-boxW / 2, -boxH / 2, boxW, boxH);
    // Small "B" label above the light

    ctx.font = 'bold 8px monospace';
    ctx.textAlign = 'center';
    ctx.fillText('B', 0, -boxH / 2 - 3);
  }
  
  ctx.restore();
}
```

Also add a short line segment from the traffic light position back toward the road to visually connect them:
```typescript
  // Connection line from light to road stop line
  ctx.beginPath();
  ctx.moveTo(stopX + perpX, stopY + perpY);
  ctx.lineTo(stopX, stopY);
  ctx.strokeStyle = tl.state === 'GREEN' ? '#00aa00' : tl.state === 'RED' ? '#aa0000' : '#aaaa00';
  ctx.lineWidth = 1.5;
  ctx.stroke();
```
Draw this BEFORE the ctx.translate/rotate block (so it's in world coordinates).

Make sure to access `tl.boxBlocked` — it's a boolean field on TrafficLightDto.
